# Contributing to Multi-Window Project Management System

Thank you for your interest in contributing! We welcome all kinds of contributions, from bug reports to new features and documentation improvements.

## Ways to Contribute

### 1. Report Bugs 🐛

If you find a bug, please open a GitHub Issue with:
- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Your environment (OS, Python version, etc.)

### 2. Suggest Improvements 💡

Have an idea? We'd love to hear it!
- Open an Issue with the `enhancement` label
- Describe the problem it solves
- Explain the proposed solution

### 3. Share Your Adaptations 📚

Used this system for your project? Share how you adapted it:
- Create a discussion or issue with your use case
- Include screenshots or examples
- Help others learn from your experience

### 4. Translate Documentation 🌍

Help make this project accessible to more people:
- Spanish (es)
- Japanese (ja)
- German (de)
- French (fr)
- Portuguese (pt)

Submit translations as PRs with:
- Translated files: README_{lang}.md, INSTALL_{lang}.md
- Updated templates with language support

### 5. Submit Code Changes 🔧

For code improvements:

**Before you start:**
- Check existing Issues and PRs to avoid duplicates
- Fork the repository
- Create a new branch: `git checkout -b feature/your-feature`

**While developing:**
- Keep changes focused and atomic
- Test with different project types
- Update documentation if needed
- Maintain bilingual (EN/ZH) documentation

**When submitting a PR:**
- Clear description of changes
- Reference related Issues
- Test locally before submitting
- Keep commits clean and organized

## Coding Standards

- Use clear, descriptive variable and function names
- Include docstrings for non-obvious code
- Keep Python compatible with 3.7+
- Test with Windows, macOS, and Linux paths

## Documentation Standards

- Keep README.md bilingual (English + 中文)
- Use clear headings and sections
- Include examples where helpful
- Link to related documentation

## Community Guidelines

- Be respectful and inclusive
- Focus on the code, not the person
- Ask questions if something is unclear
- Help others learn and improve

## Questions?

- Check [README.md](README.md) for system overview
- See [INSTALL.md](INSTALL.md) for setup help
- Open a GitHub Discussion for general questions
- Email via GitHub profile

**Thank you for making this project better!** 🙏
